# Branch Acceptance Criteria — 2026-04-13

Purpose:
- convert broad decomposition branches into auditable completion standards
- stop broad beads from being closed on partial or indirect evidence

## Doc 54 — visible output boundary
A branch is not done unless it has:
- explicit forbidden leak classes
- explicit allowed/justified reference rules
- verification checks that inspect operator-visible responses
- proof that internal routing/trace/meta state is not leaked by default

## Doc 61 — domain-general cognition core
A branch is not done unless it has:
- named primitives actually consumed by downstream branches
- at least one real consumer for each kept primitive
- no speculative primitives without a consumer or proof surface

## Doc 71 — governing priors
A branch is not done unless it has:
- bounded prior categories
- named runtime consumers
- observable trace or decision effects
- no purely rhetorical weighting constructs

## Doc 72 — identity / role / self-model
A branch is not done unless it has:
- separation between stable identity and situational role/state
- real consumer paths using those fields
- evidence that fields affect routing, permissions, or trace semantics

## Doc 73 — intention / commitment / self-regulation
A branch is not done unless it has:
- lifecycle states
- creation/persistence/release transitions
- trace visibility of transitions
- at least one consumer that changes behavior based on commitment state

## Doc 74 — identity / reference resolution
A branch is not done unless it has:
- named reference target classes
- context-specific resolution rules
- at least one slice/projection/trace consumer using the resolution logic

## Doc 75 — projection / view semantics
A branch is not done unless it has:
- canonical-vs-projected invariants
- at least one real projection consumer
- proof that projected views do not masquerade as canonical truth

## Doc 76 — retention / forgetting / decay
A branch is not done unless it has:
- explicit decay triggers
- explicit retrieval/promotion triggers
- one consumer path where retained vs decayed material affects selection
- evidence beyond local note-capping or generic decay vocabulary

## Doc 77 — governance / versioning / migration
A branch is not done unless it has:
- deprecation gates
- compatibility declarations/checks
- migration review path
- at least one exercised change/conformance path

## Doc 78 — bounded secondary cognition / persistent autonomy
A branch is not done unless it has:
- real call-site inventory
- heuristic-vs-model-backed classification
- operator-priority bounds proven in behavior
- trace/eval proof of quality and boundedness
- explicit reuse/extend/blocked/new mapping for each feature
